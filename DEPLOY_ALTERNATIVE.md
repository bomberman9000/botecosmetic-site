# Альтернативные способы деплоя

## Проблема: SSH соединение отклоняется

Если вы видите ошибку `Connection refused` при попытке подключиться к серверу:

### Вариант 1: Проверьте IP адрес сервера

Возможно, IP адрес изменился. Проверьте в панели управления вашего VPS провайдера:
- Timeweb
- Selectel  
- Другой провайдер

### Вариант 2: Используйте веб-консоль провайдера

Многие провайдеры VPS предоставляют веб-консоль (VNC/Web SSH):
1. Зайдите в панель управления вашего провайдера
2. Найдите опцию "Консоль" или "Web SSH"
3. Подключитесь через браузер
4. Выполните команды из SERVER_STEPS.txt

### Вариант 3: Используйте другой SSH порт

Если SSH работает на другом порту (не 22):
```bash
ssh -p 2222 root@144.31.64.130
# или
ssh -p 22022 root@144.31.64.130
```

### Вариант 4: Проверьте файрвол

Возможно, нужно разрешить SSH доступ через панель провайдера.

### Вариант 5: Если у вас есть доступ через панель управления

1. **Скачайте архив локально:**
   - Файл уже готов: `/Users/mac/.cursor/worktrees/1/vbg/bote-site/deploy.tar.gz`
   - Размер: ~9MB

2. **Загрузите через файловый менеджер провайдера:**
   - Зайдите в панель управления
   - Найдите файловый менеджер
   - Загрузите `deploy.tar.gz` в `/tmp/` на сервере

3. **Используйте веб-консоль для выполнения команд:**
   ```bash
   mkdir -p /var/www/bote-site
   cd /var/www/bote-site
   tar -xzf /tmp/deploy.tar.gz
   npm install
   npm run build
   pm2 delete bote-site 2>/dev/null || true
   pm2 start npm --name "bote-site" -- start
   pm2 save
   pm2 list
   ```

### Вариант 6: Подготовьте архив для ручной загрузки

Архив уже готов: `deploy.tar.gz` (9.3 MB)

Расположение: `/Users/mac/.cursor/worktrees/1/vbg/bote-site/deploy.tar.gz`

Вы можете:
- Загрузить через веб-интерфейс провайдера
- Использовать SFTP клиент (FileZilla, WinSCP)
- Использовать другой способ доступа к серверу

### Команды для выполнения на сервере (после загрузки архива)

```bash
# 1. Перейдите в директорию проекта
mkdir -p /var/www/bote-site
cd /var/www/bote-site

# 2. Распакуйте архив (если загрузили в /tmp/)
tar -xzf /tmp/deploy.tar.gz

# 3. Установите зависимости
npm install

# 4. Соберите проект
npm run build

# 5. Установите PM2 (если не установлен)
npm install -g pm2

# 6. Остановите старое приложение
pm2 delete bote-site 2>/dev/null || true

# 7. Запустите приложение
pm2 start npm --name "bote-site" -- start

# 8. Сохраните конфигурацию
pm2 save

# 9. Проверьте статус
pm2 list
pm2 logs bote-site --lines 20
```

### Проверка работы

После выполнения команд откройте в браузере:
- http://144.31.64.130:3001
